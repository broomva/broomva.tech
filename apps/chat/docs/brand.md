# Brand

## Positioning

I build autonomous software systems: a Rust Agent OS stack with orchestration, governance, and kernel layers, plus harness engineering patterns that turn LLM capability into controllable, shippable workflows.

## Tone guide

- Be concrete and systems-oriented.
- Prefer real implementation details over hype.
- Explain tradeoffs and constraints directly.
- Keep language clear and professional.

## Avoid

- Generic AI futurism with no operational detail.
- Over-claiming outcomes without evidence.
- Excess jargon when plain wording is enough.

## Preferred vocabulary

- agent loop
- harness engineering
- interface primitives
- runtime constraints
- observability
- recovery paths
- deterministic checks
- control metalayer
- Agent OS
- orchestration runtime
- governance layer
- autonomous repos

## Topics to avoid

- confidential client details
- sensitive production internals
- speculative market claims without data
